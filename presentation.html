Vad är svky.se?

En lättviktig, självhostad URL-förkortare byggd för Svenska kyrkan. Anställda kan skapa korta, minnesvärda länkar som pekar på sidor under svenskakyrkan.se — utan att behöva kontakta IT.

Tre typer av användare

  • Besökare — klickar på en kortlänk och omdirigeras
  • Slutanvändare — skapar och hanterar sina egna länkar
  • Admin — övervakar och modererar alla länkar

Grundprinciper

  • Inga lösenord — magic link via e-post
  • GDPR-vänligt — inga IP-adresser sparas
  • Endast *.svenskakyrkan.se som mål
  • 302-redirects så mål-URL:er kan ändras

Upplägg

1. Slutanvändare

Beställa, verifiera, logga in, hantera länkar, klickstatistik.

2. Admin

Alla länkar, användare, övertaganden, redigera sidor.

3. Teknik & drift

Stack, domän, SSL, server, libs, CI/CD, databas.

Varför magic link?

Inga lösenord behövs. Verifieringstokenen är en engångsbricka — used_at sätts direkt vid första anropet.

Startsidan — beställningsformulär

Anställda fyller i tre fält: egen e-post, mål-URL, önskad kod (valfritt).

[ SKÄRMDUMP: startsidan ] Ta en skärmdump av GET / (svky.se). Visa formuläret med fälten E-post, Mål-URL och Kod.

Validering

  • E-post måste sluta på @svenskakyrkan.se
  • Mål-URL måste vara HTTPS och peka på *.svenskakyrkan.se
  • Inga query strings eller fragment tillåts
  • Koden får inte vara reserverad (t.ex. admin, login)

Steg 2: "Kolla din inkorg"

Efter lyckad beställning visas en bekräftelsesida och ett verifieringsmail skickas via Lettermint SMTP.

[ SKÄRMDUMP: pending.html ] Skärmdump av sidan som visas efter POST /request. Den säger typ "Kolla din inkorg — vi har skickat en verifieringslänk".
[ SKÄRMDUMP: verifieringsmailet i inkorgen ] Öppna mailklienten och ta en bild på själva mailet ("Verifiera din kortlänk…") med knappen/länken.

Steg 3: Länken är aktiv

När användaren klickar på verifieringslänken i mailet (GET /verify/<token>) markeras länken som aktiv och en bekräftelsesida visas.

[ SKÄRMDUMP: verify_ok.html ] Ta skärmdump av sidan som visas efter verifiering — "Länken är aktiv!" med visning av den korta URL:en.

Från denna punkt fungerar kortlänken — alla klick till svky.se/<kod> omdirigeras med HTTP 302 till mål-URL:en.

Mina länkar

Efter inloggning kommer användaren till GET /my-links där alla länkar hen äger listas.

[ SKÄRMDUMP: my_links.html ] Skärmdump av /my-links med några länkar i listan. Visa gärna både aktiva och deaktiverade länkar.

Vad kan användaren göra?

  • Byta mål-URL på en befintlig kortlänk (eftersom redirects är 302 gäller ändringen direkt)
  • Avaktivera en länk (status → DISABLED_OWNER)
  • Se klickstatistik per länk
  • Begära att överföra länken till en annan kollega

Detaljvy med klickstatistik

Varje länk har en egen detaljsida där ägaren kan se klickhistorik (senaste 30 dagarna) och göra ändringar.

[ SKÄRMDUMP: my_link_detail.html ] Skärmdump av en länks detaljsida med klickgrafen (Chart.js). Visa gärna en länk med riktig klickdata.

GDPR-vänligt: endast link_id, referer och tidsstämpel sparas vid klick. Inga IP-adresser, inga cookies, ingen browser-fingerprinting.

Alla länkar

På /admin/links ser admin samtliga kortlänkar med status, ägare, antal klick och skapandedatum.

[ SKÄRMDUMP: admin/links.html ] Skärmdump av adminvyn med länktabellen. Logga in som admin och gå till /admin/links.

Härifrån kan admin snabbt klicka in sig på en specifik länk för att se detaljer eller åtgärda något.

Detaljvy för en länk

Adminns länkdetaljsida visar klickgrafen (Chart.js), audit-logg, ägare och ger möjlighet att överföra ägarskap.

[ SKÄRMDUMP: admin/link_detail.html ] Skärmdump av /admin/links/<id> — helst en länk med klick så grafen syns. Ta gärna två bilder om sidan är lång: en som visar topp (metadata + graf) och en som visar botten (audit-logg + överför ägare).

Användarhantering

/admin/users listar alla registrerade konton med e-post, antal länkar och datum för senaste inloggning.

[ SKÄRMDUMP: admin/users.html ] Skärmdump av användarlistan. Kör gärna kommandot nedan först för att säkerställa att du själv är admin:

Godkänna övertaganden

När en medarbetare begär att ta över en länk skickas ett mail till admin. Admin godkänner eller avslår direkt från mailet — en signerad token säkerställer att inga andra kan agera på begäran.

[ SKÄRMDUMP: mailet om övertagandebegäran ] Skärmdump av mailet som admin får med godkänn/avslå-knappar.
[ SKÄRMDUMP: transfer_done.html ] Bekräftelsesidan som visas efter att admin klickat på en av knapparna.

Redigerbara sidor — Markdown

Om-sidan och integritetssidan sparas i tabellen site_settings och kan redigeras av admin med Markdown + live-förhandsvisning.

[ SKÄRMDUMP: admin-redigering av Om-sidan ] Skärmdump av redigeringsvyn för Om-sidan med textrutan till vänster och förhandsvisningen till höger.

Innehållet serveras sedan via GET /om respektive GET /integritet utan att koden behöver ändras.

Del 3

Teknik & drift

Stack, server, domän, SSL och deployment.

Stack

Backend

  • Python 3.12
  • FastAPI 0.115 (ASGI)
  • uvicorn 0.30 som ASGI-server
  • Jinja2 3.1 för templates
  • itsdangerous 2.2 för signerade cookies/tokens
  • python-markdown 3.7 för redigerbara sidor
  • smtplib (stdlib) för e-post

Data & frontend

  • SQLite via sqlite3 (stdlib) — ingen ORM
  • Vanlig CSS i app/static/style.css
  • Chart.js för klickgrafer (via CDN)
  • Inga SPA-ramverk, inga byggsteg

Infrastruktur

  • Docker + Docker Compose
  • Caddy 2 som reverse proxy med auto-TLS
  • Hetzner Cloud (Helsingfors, EU)

Extern tjänst

  • Lettermint SMTP för transaktionell e-post

Två containers, ett nätverk

  • caddy — publik på port 80/443, hanterar TLS automatiskt
  • svky — FastAPI/uvicorn, lyssnar internt på 8000
  • Caddy når svky via Docker-nätverk (reverse_proxy svky:8000)
  • SQLite-databasen ligger i en volym: ./data:/app/data

Domän & SSL

Domän

  • svky.se pekas via DNS (A-record) på Hetzner-serverns publika IP
  • BASE_URL=https://svky.se i .env — används i mail-länkar

SSL/TLS

  • Caddy 2 hanterar certifikat helt automatiskt via Let's Encrypt (ACME)
  • Förnyas automatiskt i bakgrunden
  • HSTS och HTTP→HTTPS-redirect ingår by default

Hela Caddyfile:

svky.se {
    reverse_proxy svky:8000
}

Tre rader — det är allt som behövs.

Server & drift

  • Hetzner Cloud — liten VPS i datacentret i Helsingfors (Finland, inom EU)
  • OS: vanlig Linux, Docker installerat
  • Backup: SQLite-filen är en enda fil — kopieras dagligen till object storage
  • Foreign keys: PRAGMA foreign_keys = ON — schemaintegritet enforced på SQLite-nivå
  • Restart: båda containers har restart: unless-stopped

Databas­schema

Hela schemat definieras i app/database.py och skapas automatiskt vid app-start (CREATE TABLE IF NOT EXISTS).

Kärntabeller

  • users — e-post, is_admin, last_login
  • links — code, target_url, owner_id, status
  • clicks — link_id, clicked_at, referer
  • tokens — magic links (verify / login)

Stödtabeller

  • audit_log — loggar admin-åtgärder
  • rate_limits — 5 req/timme/IP/action
  • takeover_requests — övertagandebegäran
  • transfer_requests — ägaröverföring
  • site_settings — redigerbara sidor (Markdown)

Link-statusar

VärdeKonstantBetydelse
0PENDINGSkapad, väntar på e-postverifiering
1ACTIVEAktiv, omdirigerar besökare
2DISABLED_ADMINAvaktiverad av admin
3DISABLED_OWNERAvaktiverad av ägare

Designbeslut kring redirects

  • HTTP 302 — inte 301. 301 cachas permanent av webbläsaren, vilket skulle omöjliggöra ändring av mål-URL.
  • Koden är unik (UNIQUE) och indexerad för snabba lookups.

Säkerhet & integritet

Autentisering

  • Inga lösenord — endast magic links
  • Tokens är engångs (used_at sätts vid förbrukning)
  • Sessioner = signerade cookies (itsdangerous)
  • CSRF-token för alla POSTs (app/csrf.py)

Validering & gränser

  • Mål-URL måste vara HTTPS under *.svenskakyrkan.se
  • Koder valideras mot en reserverad lista
  • Rate limiting: 5 req/timme per IP per action

GDPR

  • Inga IP-adresser sparas vid klick
  • Endast e-post, länkdata och anonym klickstatistik
  • Server i EU (Hetzner, Helsingfors)

Databas

  • PRAGMA foreign_keys = ON
  • Alla lookups på code/token är indexerade

Sammanfattning

  • En liten, självhostad URL-förkortare för Svenska kyrkan
  • Python + FastAPI + SQLite — inga tunga beroenden
  • Magic link för både registrering och inloggning — inga lösenord
  • Docker + Caddy med auto-TLS — enkel drift på en liten VPS
  • GDPR-vänligt by design — inga IP-adresser, server i EU
  • CI/CD via GitHub Actions till GHCR
